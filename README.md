# 智能图色点击器 (Smart PixClick)

[![PRs Welcome](https://img.shields.io/badge/PRs-welcome-brightgreen.svg)](https://github.com/E5C8F/SmartPixClick/pulls)
> 所见即可录，所录即可点。一款基于视觉识别的桌面自动化助手。

`智能图色点击器` 是一款创新的桌面自动化工具。它摒弃了传统的坐标录制或元素监听方式，采用**窗口图色识别**技术，通过“录制-识别-点击”的直观流程，让任何屏幕上的重复操作都能轻松自动化。

下载地址：[智能图色点击器 (Smart PixClick)](https://github.com/E5C8F/SmartPixClick/blob/main/%E6%99%BA%E8%83%BD%E5%9B%BE%E8%89%B2%E7%82%B9%E5%87%BB%E5%99%A8.exe)

## ✨ 核心特性

*   **🖱️ 直观录制**：无需编写代码，简单按键即可录制屏幕识别区域与目标点击位置。
*   **🧠 智能识别**：综合匹配**窗口标题**、**区域坐标**及**像素数据**，确保识别的精准性。
*   **⚙️ 灵活执行**：提供多级别识别模式（从保守的前台点击到强制的窗口激活），适应各种复杂场景。
*   **🎛️ 容错可控**：可自定义像素颜色允许误差，有效应对画面闪烁、字体抗锯齿等微小变化。
*   **↩️ 操作可逆**：支持一键撤回最近录制，脚本管理轻松自如。

## 🚀 快速入门

### 录制你的第一个脚本

1.  **启动** 启动智能图色点击器。并在控制台直接回车以使用默认配置。
2.  **悬停**：将鼠标移动到你想让工具识别的屏幕控件上。
3.  **录制**：
    *   **按下并按住** `录制键` (默认：**数字键盘 9**)。
    *   此时，鼠标下方的区域会被记录为“识别区”。（**提示**：为防止鼠标悬停导致控件变色，可在按住按键后暂时移开鼠标）。
    *   将鼠标移动到你想点击的目标位置。
    *   **释放** `录制键`。该位置将被记录为“点击点”。
4.  **完成**：一个脚本条目已添加到列表中。当识别条件满足时，工具将自动执行点击。

### 当脚本出错时

*   按下 `撤回键` (默认：**数字键盘 0**)，即可撤销最近录制的一个脚本。

## ⚙️ 配置说明

### 脚本执行器识别级别 (0-3)

| 级别 | 模式 | 描述 |
| :--- | :--- | :--- |
| **0** | **前台识别** | 只识别和点击当前活动窗口。速度最快，干扰最小。 |
| **1** | **前台识别+窗口调整** | 同级别0，但会尝试将窗口调整为录制时的大小和状态。 |
| **2** | **后台识别与点击+窗口调整** | 对所有窗口进行后台识别和点击（可能对某些应用失效），会尝试将窗口调整为录制时的大小和状态。 |
| **3** | **强制前台识别+窗口调整** | 强制激活窗口到前台进行识别和点击，会尝试将窗口调整为录制时的大小和状态，兼容性最强。 |

### 允许误差 (0-255)

当实际屏幕像素与录制像素的RGB值之差超过此阈值时，视为不匹配。
*   **建议值**：`16`。此值能有效应对大部分抗锯齿和亮度微调，在精准度和容错性间取得良好平衡。
*   **值越小**，识别越严格。
*   **值越大**，识别越宽松。

## ❓ 常见问题

**Q: 为什么有时候录制后无法成功识别？**
**A:** 最常见的原因是鼠标悬停导致控件高亮或变色。请在按住录制键后，**将鼠标暂时移开**，以确保录制的是控件的默认状态。

**Q: 识别级别2（后台点击）在某些程序上无效？**
**A:** 这是正常现象。后台模拟点击依赖于系统的消息机制，部分程序（尤其是带反作弊系统的游戏或高权限应用）会屏蔽此类消息。

**Q: 对系统管理员权限的窗口无效怎么办？**
**A:** 工具默认以普通权限启动。如果需要对高权限窗口进行操作，请**以管理员身份运行**本工具。

**Q: 配置和录制的脚本可以保存吗？**
**A:** **目前版本暂未实现配置和脚本的持久化保存功能。** 每次启动工具都需要重新配置和录制脚本。作为替代方案，所有脚本数据的变更都会实时打印在控制台中，高级用户可以直接复制这些数据自行保存。在开发者模式下（在控制台输入 `0`），可以重新导入这些脚本数据(此操作需要相关知识，普通用户不建议使用)。

**Q: 识别级别2、识别级别3，点击A窗口的按钮打开了B窗口，但工具为什么还会不停地点击A窗口的按钮？**
**A:** 这是因为A窗口按钮的图色在点击后并未改变，工具会持续认为A窗口的触发条件仍然满足。这是基于图色识别的自动化工具的固有特性。**解决方案**：在录制此类流程时，应确保录制的图色区域在操作后会发生变化，或者识别级别设置为0、1，这样将不会识别被遮挡的窗口。

**Q: 工具安全吗？**
**A:** 本工具完全本地运行，不会将任何屏幕图像或数据上传到网络。不进行任何联网行为，因此也无法进行更新检查。
